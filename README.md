# 🇧🇷 巴西电商 Olist 业务增长与物流时效深度分析项目

## 📌 项目概述
本项目基于巴西最大集成电商平台 Olist 的真实脱敏数据集，涵盖 10 万+ 订单数据。
通过 **SQL (数据提取)** 与 **Python (探索性分析与可视化)** 的深度联动，从五个核心专题切入，还原了电商从“流量获取”到“履约交付”再到“用户留存”的全链路业务场景。

---

## 📂 模块化目录结构
遵循工业级数据分析工作流，实现代码、SQL、配置与报告的解耦。

* **`config.py`**：全局配置中心（包含 DB 连接引擎、Matplotlib 字体修复及 Seaborn 统一主题设置）。
* **`sql_scripts/`**：核心逻辑库，存储所有经过 DBeaver 调试通过的业务查询脚本。
* **`01_order_funnel_analysis.ipynb`**：专题一：订单全生命周期漏斗，评估系统健康度。
* **`02_review_logistic_analysis.ipynb`**：专题二：用户满意度归因，定位物流时效对评分的冲击。
* **`03_geo_logistics_analysis.ipynb`**：专题三：巴西地域物流差异，定位“重灾区”州。
* **`04_price_retention_analysis.ipynb`**：专题四：价格敏感度与回购特征，识别高价值 VIP 用户。
* **`05_monthly_sales_analysis`**：专题五：营收波动因素拆解，识别“黑五”等季节性增长驱动力。

---

## 📊 业务洞察与技术实现 (Insights & Tech Stack)

### 1. 履约体验归因 (专题一 & 二)
* **业务逻辑**：通过 `JULIANDAY` 计算送达耗时与预计耗时的偏差，量化物流延迟对评分的直接损伤。
* **技术重点**：利用 `ax1.twinx()` 实现双轴可视化，直观呈现“送达时间越长，评分跌落越快”的趋势。

### 2. 地域物流排行榜 (专题三)
* **业务逻辑**：针对巴西 27 个州进行时效排名。发现北部（AP, RR）与经济中心（SP）存在 2-3 倍的物流鸿沟。
* **技术重点**：使用 `HAVING total_orders > 50` 剔除样本噪声，通过 `Reds_r` 颜色渐变实现风险预警可视化。

### 3. 用户回购与价值模型 (专题四)
* **业务逻辑**：利用 `customer_unique_id` 穿透订单表，识别复购率。分析显示高频回购者对低客单价商品更有粘性。
* **技术重点**：SQL 多表下钻 (`JOIN`) 结合 Python 用户分层标签化 (User Segmentation)。

### 4. 月度营收驱动分析 (专题五)
* **业务逻辑**：拆解 GMV 增长因素。2017 年 11 月的峰值由“黑五”订单量激增驱动，而非客单价上涨。
* **技术重点**：时间序列格式化 `strftime('%Y-%m')` 结合环比增长率 `.pct_change()` 分析。

## 🛠️ 环境要求与工具
* **数据仓库**: SQLite / PostgreSQL (通过 SQL 脚本直接运行)
* **分析语言**: Python 3.10+
* **核心库**:

    * `Seaborn`: 职业化图表绘制（已适配 v0.14.0 调色盘规范）
    * `Matplotlib`: 复杂图形控制与多坐标轴实现
* **版本管理**: Git (包含针对数据库大文件的 `.gitignore` 配置)
